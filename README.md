#### 第 1 套：入门级（MyBatis-Plus 环境搭建与基础 CRUD）

##### 一、业务场景
公司准备开发一个“员工管理”模块，需要先完成 MyBatis-Plus 的基础集成，用于后续员工数据的增删改查。
本题重点不是复杂业务，而是验证你是否能够从零完成 MyBatis-Plus 的基础接入，并使用其通用能力完成最基本的数据操作。
##### 二、题目要求
1. 创建一个 Spring Boot 项目，并集成 MyBatis-Plus；
2. 配置数据库连接信息，数据库可使用 MySQL；
3. 创建员工表 emp，字段至少包含：
  - id
  - name
  - age
  - salary
  - dept_id
  - create_time
4. 编写对应的实体类 Emp；
5. 编写 EmpMapper，继承 MyBatis-Plus 提供的基础 Mapper；
6. 完成以下基础操作并测试：
  - 新增 1 条员工数据
  - 根据 id 查询员工
  - 根据 id 修改员工工资
  - 根据 id 删除员工
7. 使用测试类或接口方式验证以上功能；
8. 简要说明 MyBatis-Plus 相比原生 MyBatis 在开发上的便利性。

#### 第 2 套：初级（Mapper 层实现与常用业务方法）

##### 一、业务场景
员工管理模块完成基础集成后，业务层需要进一步使用 MyBatis-Plus 提供的通用 Service 方法，快速完成批量保存、列表查询、更新与删除等常见业务操作，减少重复开发工作量。
二、题目要求
1. 基于 emp 表继续开发；
2. 编写 EmpService 和 EmpServiceImpl，继承 MyBatis-Plus 提供的通用 Service；
3. 使用业务层方法完成以下操作：
  - 保存单个员工
  - 批量保存至少 3 条员工数据
  - 查询全部员工列表
  - 根据 id 查询员工
  - 根据 id 修改员工年龄或工资
  - 根据 id 删除员工
4. 至少演示以下常见业务方法中的若干个：
  - save
  - saveBatch
  - getById
  - list
  - updateById
  - removeById
5. 编写测试代码或接口验证各方法执行结果；
6. 简要说明 Mapper 层与 Service 层在职责上的区别。

#### 第 3 套：中级（条件查询）

##### 一、业务场景
员工管理模块上线后，前端需要支持多条件筛选员工数据，例如按姓名模糊查询、按年龄范围查询、按部门查询、按工资区间查询等。
为了避免手写大量 SQL，要求使用 MyBatis-Plus 提供的条件构造器完成动态条件查询。
##### 二、题目要求
1. 基于 emp 表准备至少 8 条测试数据，覆盖不同部门、年龄、工资；
2. 使用 QueryWrapper 或 LambdaQueryWrapper 实现以下查询场景：
  - 根据姓名模糊查询员工
  - 查询年龄大于等于某值的员工
  - 查询工资在某区间范围内的员工
  - 查询指定部门下的员工
  - 查询姓名不为空的员工
  - 查询多个部门中的员工
3. 增加一个组合查询场景，例如：
  - 姓名模糊匹配 + 年龄范围 + 部门筛选
4. 要求动态条件合理处理，例如：
  - 查询参数为空时，不拼接对应条件
5. 编写测试代码或查询接口验证以上功能；
6. 简要说明 LambdaQueryWrapper 相比普通字符串字段写法的优点。

第 4 套：中高级（分页查询 + 排序 + 投影查询）

#### 一、业务场景
后台管理系统中的员工列表页，需要支持分页展示、按工资排序、按创建时间倒序查看，同时为了减少无用字段传输，列表页只要求返回部分字段，例如员工编号、姓名、工资。
请基于 MyBatis-Plus 完成分页、排序与投影查询功能。
##### 二、题目要求
1. 基于 emp 表准备不少于 15 条测试数据；
2. 配置 MyBatis-Plus 分页插件；
3. 实现员工分页查询，要求支持以下参数：
  - 当前页
  - 每页条数
4. 在分页基础上增加排序功能：
  - 按工资升序
  - 按创建时间降序
5. 实现投影查询，只查询以下字段：
  - id
  - name
  - salary
6. 增加一个组合场景：
  - 按部门筛选 + 分页 + 工资排序 + 只返回部分字段
7. 编写测试代码或接口验证功能；
8. 简要说明分页查询和普通列表查询在使用场景上的区别。
